React JS

React可以使用JSX文件来写表现层，这意味着我们可以把JavaScript和HTML文件放在一个文件中。这是一个需要花一点时间去适应的过程，所以在这个教程中我们没有使用JSX来写，但是我们仍然推荐大家使用JSX，因为它可以更直观的看出哪个组件需要渲染。

Props

那么当我们准备传一些数据到我们的React class中时应该怎么做呢？上面那个例子没有提到任何相关的机制，这里提供一个。

这里我们引入一个props参数在我们的React class中，用来指明传送什么数据到react的view层。props变量的任何变化都会触发一个在view层某个位置的重绘。这些变化可能来自这个view的自身或者是一个父view，也或是它拥有的任何子view。这个方法非常好用，因为它允许我们分发数据在多个view中并且让它们保持同步。在这个代码片段中，我传了我的名字到HelloWord class中，它会用这个props变量重绘div的内部。

State

这个代码片段引入了react class中另一个部份：state。一个react类的state属性允许我们跟踪view的内部变化。如同props，任何变化都会触发view中对应元素的重绘，附带上一个条件：我们必须调用一个setState的方法，正如例子中的increment function。 永远记得使用setState，否则，你的变化不会改变到你的view，这是使用react最重要的一件事。当使用内部状态时需要getInitialState function，它用来描述view初始的样子应该是什么样的。即使你的状态初始从完全没有开始的也确保包含这个方法，因为一个空的状态也是一个状态。

那么什么时候使用props或者state或者只是用私有变量呢。Props是用来在父子classes中传递数据的，任何对一个prop的变化都会引发一个自动的view重绘，包括父view和子view。对于那些只关联到view的数据，使用state，state的任何变化也会重绘view。对于只与class有关系而不包括view的，你可以使用私有变量，但是我仍然推荐使用state；因为（追踪变化）正是它的真正目的。
